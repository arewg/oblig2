Git & GitHub

Hva er git?

Git er et kontrollsystem som holder kontroll over versjonene av filer. Dette verktøyet gjør at du alltid kan gå tilbake til tidligere versjoner av filene du jobbet på om du skulle gjøre en endring og noe feilet. En annen smart ting med git er at det lar flere mennesker jobbe sammen på de samme filene, og programmet vil notere seg hvem som gjorde hvilke forandringer, og når de gjorde det. Uten dette verktøyet kan man fort miste oversikt over når en feil ble gjort da man ikke kan gå stegvis tilbake, og det ville gjort det utrolig vanskelig for flere mennesker å jobbe sammen på et dokument uten å vente på tur til noen har gjort en endring og har sendt dokumentet videre til deg.

GitHub

GitHub er et nettsted hvor man kan laste opp, oppbevare og dele arbeidet sitt – men det er flere nyttige funksjoner enn dette. Forking er en av disse, hvor man kan opprette et nytt prosjekt basert på andres prosjekter og jobbe og videreutvikle andres ideer. Blir orginalprosjektet oppdatert kan dette enkelt implementeres i din versjon også. Gjør du en god jobb med «forkingen» og ønsker at de som først opprettet prosjektet skal se din versjon kan du gjøre det som kalles «pull request». Her gir GitHub deg muligheten til å kommunisere med utvikleren.

GitHub fungerer nesten som en slags CV også, for når man har opprettet en bruker vil din profil vise hva du har vært med på å utvikle og hvor du har hjulpet andre, på denne måten kan man andre se hvor god du er, og man utvikler et nettverk. «Changelog»-funksjonen holder tritt med hvem, hva og hvor det skjer endringer i prosjektene, noe som fort kan bli forvirrende når man er mange som samarbeider og skal holde styr på dette selv.

Selv synes jeg dette er et utrolig verktøy som gjør at man kan utnytte det fulle potensialet i ethvert prosjekt, og man kan finne andre man ønsker å samarbeide med, kan finne inspirasjon hos eller andre prosjekter man kan gjøre endringer på for å øve seg selv, og med Pull Request funksjonen kan man muligens også få en tilbakemelding!
